  ==========================================================
  1. OVERVIEW
  ==========================================================

  This project modernizes the classic 4images v1.10 gallery
  while preserving the original design 1:1.

  IMPORTANT: This is NOT a redesign! The visual layout remains
  identical - only the code is being modernized.


  ==========================================================
  2. MODERNIZATION PHILOSOPHY
  ==========================================================

  "1:1 Modernization" means:

  ‚úÖ KEEP:
     - Original layout (column count, structure, positions)
     - Color scheme (only slightly optimized)
     - All functions (upload, categories, gallery, admin)
     - License terms (4images copyright in footer)

  ‚úÖ MODERNIZE:
     - Table layouts ‚Üí Flexbox/Grid (CSS)
     - Inline styles ‚Üí Tailwind CSS classes
     - Fixed width ‚Üí Responsive design
     - GIF borders ‚Üí CSS borders/shadows
     - Old icons ‚Üí FontAwesome 7.0
     - PHP 5.x ‚Üí PHP 8.4+ (planned)

  ‚ùå DO NOT CHANGE:
     - Visual appearance (must look identical!)
     - Functional logic (remains the same)


  ==========================================================
  3. TECH STACK
  ==========================================================

  FRONTEND:
  ---------
  - Tailwind CSS 3.x (via Play CDN)
    URL: https://cdn.tailwindcss.com
    ‚Üí NO npm, NO build process!

  - FontAwesome 7.0.0 Free (via CDN)
    URL: https://cdnjs.cloudflare.com/ajax/libs/font-awesome/7.0.0/css/all.min.css
    ‚Üí Icons for navigation, buttons, etc.

  - Vanilla JavaScript ES6+
    ‚Üí No jQuery, no frameworks
    ‚Üí Only for mobile menu toggle

  BACKEND:
  --------
  - PHP 7.4+ (current)
  - PHP 8.4+ (planned for Task 1)
  - MySQL 5.7+ / MariaDB 10.3+
  - Original 4images template system (.html files)

  RESPONSIVE DESIGN:
  ------------------
  - Mobile: < 640px
  - Tablet: 640px - 1024px
  - Desktop: > 1024px
  - Max-Width: 1152px (max-w-6xl container)
  - Approach: Mobile-first

  5. LAYOUT STRUCTURE
  ==========================================================

  HEADER (header.html):
  ---------------------
  - Sticky header (stays on top when scrolling)
  - Logo + site name
  - Desktop: Horizontal navigation
  - Mobile: Hamburger menu
  - Search field (visible on desktop, in menu on mobile)

  FOOTER (footer.html):
  ---------------------
  - 3-column grid (desktop)
  - 1-column (mobile)
  - Columns: About, Quick Links, User Area
  - Copyright notice (REQUIRED by license!)

  MAIN LAYOUT (home.html example):
  ---------------------------------
  Desktop:
    [ Sidebar (3/12) | Content (9/12) ]

  Mobile:
    [ Sidebar (full-width) ]
    [ Content (full-width) ]

  Implemented with CSS Grid:
    <div class="grid grid-cols-1 lg:grid-cols-12 gap-6">
      <aside class="lg:col-span-3">...</aside>
      <section class="lg:col-span-9">...</section>
    </div>

  ==========================================================
  10. DEPLOYMENT
  ==========================================================

  REQUIREMENTS:
  -------------
  - Web hosting with PHP 7.4+
  - MySQL 5.7+ or MariaDB 10.3+
  - NO special Node.js/npm requirements!
  - NO build processes!

  WHAT GETS UPLOADED:
  -------------------
  - All PHP files (as before)
  - Modernized .html templates
  - Original data/, includes/, lang/ folders
  - config.php (created during installation)

  WHAT DOES NOT GET UPLOADED:
  ----------------------------
  - .vscode/, .claude/ (dev tools)
  - CLAUDE.md, REDESIGN_PLAN.md (development docs)
  - .git/ (Git history)
  - Dockerfile.dist, docker-compose.yml.dist

  CDN CONNECTIONS:
  ----------------
  Server must allow outgoing HTTPS connections to:
  - cdn.tailwindcss.com (Tailwind CSS)
  - cdnjs.cloudflare.com (FontAwesome)

  If CDN access is not possible:
  ‚Üí Tailwind CSS and FontAwesome can be hosted locally
  ‚Üí See https://tailwindcss.com/docs/installation

  12. PERFORMANCE
  ==========================================================

  ADVANTAGES:
  -----------
  + CDN caching for Tailwind/FontAwesome
  + Less custom CSS ‚Üí smaller load times
  + System fonts ‚Üí no web font download
  + No GIF borders ‚Üí fewer HTTP requests

  DISADVANTAGES:
  --------------
  - Initial CDN connection required
  - Tailwind Play CDN larger than custom CSS
    (For production: Tailwind CLI with purge recommended)

  ==========================================================
  15. LESSONS LEARNED
  ==========================================================

  1. Tables ‚Üí Divs:
     Always replace old <table> structures with modern CSS Grid/Flexbox
     for better responsiveness.

  2. Modernize colors:
     Yellow from #fcdc43 ‚Üí #fbbf24 (less bright, more modern)

  3. Responsive container:
     max-w-6xl (1152px) instead of fixed width for better UX

  4. Remove GIF borders:
     Replace all border*.gif with CSS borders/shadows

  5. Sticky header:
     Header with sticky top-0 z-50 for better navigation

  6. Mobile menu:
     Hamburger menu with vanilla JS toggle (no jQuery)

  7. Tailwind nested config:
     brand.blue instead of brand-blue for better organization

  8. System fonts:
     Native font stack for faster loading

  9. Git cleanup:
     Exclude dev files via .gitignore, don't rewrite history!

  10. Documentation:
      Always keep CLAUDE.md, README.md and docs/ up to date
